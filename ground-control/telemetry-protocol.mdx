---
title: "Telemetry protocol"
description: Complete reference for all UDP messages exchanged between the rocket, launcher, and dashboard.
---

The system uses a three-node communication chain. The rocket talks to the launcher over a wired UART link; the launcher relays data to the dashboard over WiFi UDP.

```
Rocket  <──UART Serial2 115200──>  Launcher  <──UDP WiFi 192.168.4.1:4444──>  Dashboard
```

## Transport layers

| Link | Protocol | Speed / address |
|------|----------|----------------|
| Rocket ↔ Launcher | UART (Serial2, GPIO 16/17) | 115200 baud |
| Launcher ↔ Dashboard | UDP over WiFi SoftAP | `192.168.4.1:4444` |

---

## Rocket → Launcher (UART)

Messages sent by the rocket over Serial2.

| Message | Format | Interval | Description |
|---------|--------|----------|-------------|
| DATA | `DATA,<ax>,<ay>,<az>,<roll>,<rate>,<servo_offset>,<state>,<Kp>,<Kd>,<skew>` | 50 ms | Full telemetry packet with IMU, roll stabilization state, and active PID gains |
| READY | `READY` | 1000 ms | Sent while in IDLE state to signal the rocket is alive and waiting |
| IGNITED | `IGNITED` | Once | Sent when the ignition servo retracts and state transitions to FLIGHT |

---

## Launcher → Rocket (UART)

Commands sent by the launcher over Serial2.

| Command | Description |
|---------|-------------|
| `ARM` | Arms the rocket; also triggers an internal gyro calibration on the rocket |
| `IGNITE` | Starts the ignition sequence (deploys ignition servo) |
| `CALIBRATE` | Re-zeroes the gyro and resets roll angle without changing arm state |
| `PID,<Kp>,<Kd>` | Updates proportional and derivative gains at runtime |

---

## Dashboard → Launcher (UDP)

Commands sent by the dashboard to `192.168.4.1:4444`.

| Command | Format | Description |
|---------|--------|-------------|
| Keepalive | `HELLO` | Discovery and keepalive; sent every 2 s. Launcher records the sender's IP as the dashboard address. |
| Launch | `launch` | Sets `udpLaunchTriggered` flag on the launcher, which then sends `CALIBRATE` + `IGNITE` to the rocket |
| Calibrate | `calibrate` | Relayed directly as `CALIBRATE` to the rocket over UART |
| PID update | `PID,<Kp>,<Kd>` | Relayed directly as-is to the rocket over UART (e.g. `PID,0.8,0.3`) |

---

## Launcher → Dashboard (UDP)

Messages sent by the launcher to the dashboard's IP on port 4444.

| Message | Format | Interval | Description |
|---------|--------|----------|-------------|
| Telemetry | `T,<ms>,<roll>,<rate>,<output>` | 50 ms | Roll telemetry extracted from the rocket's DATA packet |
| Status | `STATUS:<state>,<Kp>,<Kd>,<skew>` | 50 ms | Rocket state and active PID gains |
| Environment | `ENV,<lat>,<lon>,<alt>,<gpsState>` | 1000 ms | GPS coordinates and EMA-filtered barometric altitude |
| Fusion | `[FUSION] Hdg: <heading> \| Pitch: <pitch>` | 500 ms | Compass heading from the launcher's QMC5883L (received but ignored by the dashboard) |

<Note>
The `T,` and `STATUS:` messages are generated by the launcher at 50 ms intervals by splitting the rocket's `DATA,` UART packet — the launcher does not forward the raw `DATA,` packet to the dashboard.
</Note>

---

## Field definitions

| Field | Type | Description |
|-------|------|-------------|
| `<ms>` | integer | `millis()` timestamp on the launcher at time of forwarding |
| `<roll>` | float | Integrated roll angle in degrees (zeroed at calibration) |
| `<rate>` | float | Gyro roll rate in degrees/second |
| `<output>` / `<servo_offset>` | integer | PD controller output, constrained to ±12 |
| `<state>` | string | One of: `IDLE`, `ARMED`, `IGNITING`, `FLIGHT` |
| `<Kp>` | float | Active proportional gain |
| `<Kd>` | float | Active derivative gain |
| `<skew>` | float | `physical_skew_angle` in degrees — launcher tilt measured at last calibration |
| `<ax>`, `<ay>`, `<az>` | float | Raw accelerometer readings (m/s²) from the rocket's MPU6050 |
| `<lat>`, `<lon>` | float | GPS coordinates; `0.000000` when no fix |
| `<alt>` | float | EMA-filtered barometric altitude in meters ASL |
| `<gpsState>` | integer | `0` = no NMEA data, `1` = searching for fix, `2` = fix acquired |

---

## Example packets

```text
T,12345,3.21,-1.84,2
STATUS:FLIGHT,0.50,0.20,1.30
ENV,34.052300,-118.243700,245.3,2
PID,0.8,0.3
```

The first two lines arrive together every 50 ms during flight. `ENV` arrives every 1000 ms. `PID` is only sent when the operator clicks **UPLOAD NEW PID** in the dashboard.
