## Overview

The launcher is an ESP32-based ground station that arms and fires the rocket, relays telemetry to the dashboard, and provides GPS position, compass heading, and barometric altitude. It creates a WiFi access point that the dashboard connects to over UDP.

CAD file (Fusion 360 archive format):

- `CAD Files/Launcher Platform.f3z` — launcher platform and mount assembly

## Electronics

| Component | GPIO / Interface | Notes |
|---|---|---|
| ESP32 DevKit V1 | — | Ground station controller |
| QMC5883L compass | SDA=21, SCL=22 | I2C 0x0D, heading |
| BMP180 barometer | SDA=21, SCL=22 | I2C 0x77, altitude |
| GPS module | GPIO 4 (RX1) | Serial1 at 9600 baud, NMEA |
| Launcher servo | GPIO 19 | Ignition relay; ON=170°, OFF=55° |
| Arm switch | GPIO 5 | `INPUT_PULLUP`; LOW = armed |
| Launch button | GPIO 18 | `INPUT_PULLUP`, N.O., active LOW |
| LED indicator | GPIO 23 | Status LED |
| Buzzer | GPIO 2 | Active buzzer, LOW-trigger |
| UART to rocket | TX2=GPIO 17, RX2=GPIO 16 | Serial2, 115200 baud, 2048-byte RX buffer |

<Note>
The QMC5883L and BMP180 share the I2C bus at SDA=GPIO 21, SCL=GPIO 22. The bus runs at 400 kHz. The launcher does not have an onboard MPU6050 — accelerometer data used for tilt-compensated heading is sourced from the rocket's IMU via the UART telemetry stream.
</Note>

<Warning>
The values above come from `Launcher/src/main.cpp` and are authoritative. `docs/WIRING.md` lists different pin assignments for several signals (GPS RX, arm switch, launch button, LED, buzzer, and ignition servo) — do not rely on that file.
</Warning>

## WiFi configuration

The launcher creates a software access point on boot. The dashboard and any other clients connect to this network.

| Parameter | Value |
|---|---|
| Mode | SoftAP (access point) |
| SSID | `ROCKET_LAUNCHER` |
| IP address | 192.168.4.1 |
| UDP port | 4444 |

## Sensor fusion

### Compass and tilt compensation

The launcher combines the QMC5883L magnetometer with pitch and roll angles derived from the rocket's MPU6050 accelerometer data (forwarded in `DATA` telemetry packets over UART) to compute a tilt-compensated heading:

```
pitch = atan2(-g_fwd, sqrt(g_right² + g_down²))
roll  = atan2(g_right, g_down)

x_m = cal_y·cos(pitch) + cal_z·sin(roll)·sin(pitch) + cal_x·cos(roll)·sin(pitch)
y_m = cal_z·cos(roll) - cal_x·sin(roll)

heading = atan2(y_m, x_m) × (180 / π)   [0–360°]
```

Raw magnetometer readings are calibrated before use:

| Axis | Offset | Scale |
|---|---|---|
| X | -211.00 | 0.988 |
| Y | 897.00 | 0.971 |
| Z | -514.50 | 1.044 |

Applied as: `cal_axis = (raw - offset) × scale`

<Tip>
If the compass returns all-zero readings the firmware automatically re-initializes the I2C bus and compass without halting. Check physical connections if this happens repeatedly.
</Tip>

### Altitude filtering

BMP180 altitude readings are smoothed with an exponential moving average (EMA):

```
filteredAlt = (0.15 × newAlt) + (0.85 × filteredAlt)
```

Alpha = **0.15** — lower values produce a smoother output at the cost of slower response to rapid altitude changes.

## GPS startup

On boot the launcher checks for valid NMEA data from the GPS module for up to **5 seconds**. If no `$` character (start of an NMEA sentence) is received within that window, the launcher halts and enters an error loop:

- LED flashes continuously
- Buzzer plays a 5-beep error tone on repeat

The launcher will not proceed past startup until a GPS module is connected and sending data.

<Warning>
The GPS module is receive-only from the ESP32's perspective. Connect only the GPS TX line to GPIO 4 (ESP32 RX1). The GPS TX pin in `SerialGPS.begin` is set to `-1`.
</Warning>

## Assembly

<Steps>
  <Step title="Print the platform">
    Print the launcher platform from `CAD Files/Launcher Platform.f3z`.
  </Step>
  <Step title="Mount the ESP32 and sensors">
    Secure the ESP32 DevKit V1, MPU6050, QMC5883L, and BMP180 to the platform. Connect all three sensors to the shared I2C bus (SDA=GPIO 21, SCL=GPIO 22).
  </Step>
  <Step title="Wire the GPS module">
    Connect only the GPS module TX line to GPIO 4. Leave the GPS RX line disconnected — the launcher sends no commands to the GPS.
  </Step>
  <Step title="Install controls and indicators">
    Connect the arm switch to GPIO 5 (active LOW with internal pull-up), the launch button to GPIO 18 (active LOW, N.O.), the status LED to GPIO 23, and the active buzzer to GPIO 2.
  </Step>
  <Step title="Install the launcher servo">
    Mount the ignition relay servo on GPIO 19. Verify it travels freely between 55° (safe) and 170° (fire).
  </Step>
  <Step title="Connect the rocket UART cable">
    Wire TX2 (GPIO 17) and RX2 (GPIO 16) to the launch tube umbilical connector. TX on the launcher connects to RX on the rocket, and vice versa.
  </Step>
</Steps>
